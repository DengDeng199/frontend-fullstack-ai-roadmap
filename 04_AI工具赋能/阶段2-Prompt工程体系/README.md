## 学习内容

### 2.1 Prompt 基础结构

一个优秀的 Prompt 包含以下要素：

```
1. 角色设定（Role）     — "你是一个 Vue 3 前端专家"
2. 上下文（Context）    — 引用相关文件/文档/项目背景
3. 任务指令（Task）     — "实现一个带排序功能的表格组件"
4. 约束条件（Constraints） — "使用 Composition API + TypeScript"
5. 输出格式（Format）   — "输出完整的 .vue 文件"
6. 示例（Examples）     — "参考这个组件的风格：@Files (UserList.vue)"
```

### 2.2 高级 Prompt 技巧

**分步推理（Chain of Thought）**：
```
请一步步思考：
1. 先分析这个需求的核心功能
2. 设计数据结构和接口
3. 再实现代码
```

**思维树（Tree of Thought）**：
```
请给出至少 3 种实现方案：
方案A：...
方案B：...
方案C：...
然后分析各方案的优劣，推荐最佳方案。
```

**自我审查与修正**：
```
请生成代码后自行审查：
1. 检查是否有潜在的 bug
2. 检查是否处理了边界情况
3. 如果发现问题，请修正并说明
```

**元提示（Meta-Prompting）**：
```
你需要我提供什么额外信息才能给出最佳答案？
```

**少样本学习（Few-shot）**：
```
参考以下代码风格：
输入示例1: "xxx" → 输出示例1: "yyy"
输入示例2: "xxx" → 输出示例2: "yyy"
现在处理：输入: "zzz" → 输出: ?
```

### 2.3 编码场景 Prompt 模板库

**功能开发 Prompt 模板**：
```
角色：你是一个 {framework} {expert}。
上下文：当前项目使用 {tech_stack}。
任务：实现 {feature_description}。
约束：
- 使用 {api_style}
- 遵循 {code_style}
- 处理 {edge_cases}
输出：完整的 {file_type} 文件，包含必要的类型定义和注释。
```

**代码审查 Prompt 模板**：
```
请审查以下代码，从以下维度给出反馈：
1. 逻辑正确性 — 是否有 bug
2. 边界情况 — 是否处理了 null/undefined/空数组等
3. 性能 — 是否有性能问题
4. 安全 — 是否有安全漏洞
5. 可读性 — 命名/注释是否清晰
6. 最佳实践 — 是否遵循框架最佳实践

{code_content}
```

**Bug 排查 Prompt 模板**：
```
我在 {file} 中遇到以下错误：

错误信息：
{error_message}

相关代码：
{code_snippet}

请帮我分析可能的原因，并给出修复方案。
```

**重构优化 Prompt 模板**：
```
请帮我优化以下代码：
1. 提高可读性（提取函数/简化逻辑）
2. 提高可维护性（减少重复代码）
3. 如果可能，提高性能

优化后的代码需要保持相同的功能行为。

{code_content}
```

**文档生成 Prompt 模板**：
```
请为以下 {component/hook/function} 生成使用文档：
1. 功能说明
2. Props/参数说明
3. 使用示例（至少 3 个常见场景）
4. 注意事项

{code_content}
```

### 2.4 AI 代码审查能力

**审查 AI 生成代码的正确性**：
- 是否正确使用了框架 API
- 是否有逻辑错误
- 是否处理了异步操作
- 是否有类型错误

**常见 AI 编码错误类型**：
- 幻觉（使用了不存在的 API）
- 过时的 API（使用了旧版本）
- 安全漏洞（XSS / 注入）
- 性能问题（不必要的重渲染/内存泄漏）
- 过度工程化（简单问题复杂化）

**如何让 AI 生成更可靠的代码**：
- 提供框架版本号
- 引用官方文档
- 给出具体约束
- 要求 AI 自审
- 分步生成和验证

### 2.5 Prompt 效果评估

- **A/B 测试不同 Prompt**：同一个任务，用不同 Prompt 对比输出质量
- **量化评估指标**：
  - 一次通过率（不需要修改即可使用）
  - 修改次数（需要几次迭代）
  - 准确率（功能是否正确）
- **持续优化模板**：根据实际效果不断调整 Prompt 模板
