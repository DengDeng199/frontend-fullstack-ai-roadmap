## 学习内容

### 1.1 Cursor IDE（推荐）

**安装与配置**：
- 安装 Cursor IDE（cursor.com）
- 导入你的项目设置（字体/主题/快捷键）
- 配置 Cursor Rules（项目级提示规则）

**核心操作**：
- **Ctrl+K** — 行内编辑（选中代码 → 输入指令 → AI 直接修改）
- **Ctrl+L** — 聊天面板（提问/解释/生成代码）
- **@ 符号引用**：
  - `@Files` — 引用项目文件
  - `@Folders` — 引用整个目录
  - `@Web` — 引用网页内容
  - `@Docs` — 引用框架文档
- **Composer（多文件编辑）**：跨文件同时修改
- **Tab 自动补全**：智能代码补全（类似 Copilot）
- **Ctrl+/** — 触发 AI 编辑建议

### 1.2 GitHub Copilot（如使用 JetBrains）

**安装**：
- WebStorm / IntelliJ IDEA 安装 Copilot 插件
- 登录 GitHub 账号

**核心操作**：
- 行内代码补全（灰色建议 → Tab 接受）
- Copilot Chat（侧边栏对话）
- 代码解释 / 生成测试 / 修复 Bug

### 1.3 其他工具了解

- **Windsurf** — 类似 Cursor 的 AI IDE
- **Claude Code** — 终端 AI 编程助手
- **WorkBuddy Code** — AI 辅助开发工具

### 1.4 有效使用技巧

**提供充足上下文**：
```
# 不好
"帮我写一个表格组件"

# 好
"@Files (Table.vue) 我需要在这个表格组件中增加排序功能，
参考 @Docs (Element Plus Table) 的 sortable 属性，
使用方式和我们项目中 UserList.vue 中的排序一致"
```

**分步指令**：
```
# 不好
"帮我重构这个页面，加上暗色模式，优化性能，写单测"

# 好（分步）
"第一步：先帮我把这个组件的样式提取为 CSS 变量"
→ 确认后再继续下一步
```

**代码审查习惯（非常重要）**：
- AI 生成的代码必须审查，不能直接用
- 检查逻辑正确性
- 检查是否有安全漏洞
- 检查是否符合项目代码规范
- 检查是否引入不必要的依赖

**逐步迭代**：
```
"先生成基础骨架" → 审查 → "加上错误处理" → 审查 → "加上类型定义" → 最终版
```

### 1.5 Cursor Rules 配置示例

```
你是一个 Vue 3 前端开发专家。
- 使用 Vue 3 Composition API + <script setup>
- 使用 TypeScript
- 使用 Element Plus UI 框架
- 使用 Pinia 状态管理
- 组件命名使用 PascalCase
- 样式使用 SCSS
- 遵循项目现有代码风格
```
