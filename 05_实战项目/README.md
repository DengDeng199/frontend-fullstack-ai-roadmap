# 实战项目规划

> 所有学习的最终目标是产出价值。每个阶段的实战产出都在这里管理和追踪。
> 项目可以同时在多个方向上推进，优先选择能直接服务公司业务的场景。

---

## 一、项目优先级原则

1. **公司业务优先** — 在 caidiaweb / odvp-web / bcs-web 中落地的学习最有价值
2. **可见性优先** — 能让团队/领导看到产出的学习最容易获得支持
3. **渐进式推进** — 从小工具开始，逐步做大型项目
4. **文档记录** — 每个项目完成后都写总结文档存入笔记文件夹

## 三、项目执行建议

1. **每个项目建立独立子文件夹**，放在对应阶段的「笔记」目录下
2. **项目启动前写简短方案**（1 页即可），明确范围和产出
3. **完成后写总结文档**：做了什么、学到什么、遇到什么坑
4. **优先在公司项目中落地**，做到"学习即工作"
5. **定期回顾**，每月检查一次进度，调整优先级
