# 二、V8 垃圾回收机制（核心面试重点）

## 1\. 分代回收思想

V8 将堆内存分为两块，**根据对象存活时间采用不同算法**：

- **新生代**：短期存活对象（临时变量、函数局部对象），占内存小

- **老生代**：长期存活对象（全局变量、闭包、缓存数据），占内存大

## 2\. 新生代 GC：Scavenge 复制清除算法

### 原理

新生代内存分为两个等大空间：**From（使用区） \+ To（空闲区）**

1. 新对象全部分配在 From 空间

2. GC 触发时：**存活对象复制到 To 空间**

3. 直接清空整个 From 空间

4. 互换 From / To 角色，完成回收

### 特点

- 速度极快，适合大量短期死亡对象

- **空间换时间**，浪费一半内存

- 存活多的对象会**晋升到老生代**

### 案例

```js

// 函数执行完立即销毁 → 新生代快速回收
function temp() {
  let arr = new Array(1000) 
}
temp()
// arr 瞬时存活，Scavenge 快速清理

```

## 3\. 老生代 GC：标记清除 / 标记整理

老生代对象存活久、数量大，不适合复制算法，使用两种组合算法：

### （1）标记清除 Mark\-Sweep

- 标记：遍历所有存活对象打标

- 清除：删除未标记垃圾对象

- **缺点**：产生大量内存碎片

### （2）标记整理 Mark\-Compact

- 标记流程一致

- 不直接删除，**将存活对象向内存一侧平移整理**

- 彻底消除内存碎片

- 速度慢，偶尔执行

# 三、V8 如何解决 GC 卡顿（STW 优化）

**STW（Stop The World）**：GC 执行时暂停 JS 主线程，造成卡顿、掉帧

## 1\. 增量标记 Incremental Marking（核心）

**旧方案**：一次性标记全部老生代对象，STW 时间极长

**优化方案**：把标记过程拆分成「小段」，**穿插在 JS 执行间隙**

- 执行一段 JS → 标记一点垃圾 → 继续执行业务

- 大幅降低单次 STW 时间，解决页面卡顿

## 2\. 并行回收 Parallel

GC 标记阶段开启多线程并行执行，**多条线程同时标记垃圾**，主线程短暂等待，速度大幅提升。

## 3\. 并发回收 Concurrent

**GC 线程在 JS 主线程运行的同时，后台并发标记**

几乎不阻塞主线程，是目前 V8 最极致的 GC 优化

> **面试标准答案**：V8 通过 增量标记 \+ 并行标记 \+ 并发标记，将「整段长时间STW」拆分为「极短多次STW」，彻底解决大内存页面卡顿。

# 四、隐藏类 Hidden Class（JS 对象提速核心）

## 1\. 背景问题

JS 是**动态弱类型**，对象属性可以随意增删，引擎无法预知结构，查找极慢。

## 2\. 原理

V8 为每一个**结构相同的对象**创建一个 **隐藏类（Descriptor）**

- 存储对象：属性名、顺序、偏移量、类型

- 相同结构对象复用同一个隐藏类

- 读取属性直接按偏移量读取，不再遍历查找

## 3\. 结构变更 = 隐藏类跳转

```js

// 对象结构一致 → 同一个隐藏类
const obj1 = { a: 1, b: 2 }
const obj2 = { a: 10, b: 20 }

// 新增属性 → 生成新隐藏类（降级性能）
obj1.c = 3 

```

## 4\. 开发最佳实践（高频优化）

- **尽量统一对象结构**

- 避免运行时随意新增/删除属性

- 避免属性顺序错乱

# 五、内联缓存 Inline Cache（IC）

## 1\. 核心作用

**缓存属性查询结果**，让重复读取对象属性接近「原生变量速度」

## 2\. 原理

V8 发现代码多次读取同一对象属性时：

1. 记录当前对象的隐藏类

2. 缓存该属性的内存偏移地址

3. 下次读取直接取值，不查找、不遍历

## 3\. 为什么代码写规范能提速？

频繁变更对象结构 → **IC 缓存失效** → 性能断崖式下跌

## 4\. 实战案例

```js

// 结构稳定 → IC 缓存命中，极速执行
function getAge(user) {
  return user.age
}

// 多次调用相同结构对象，IC 持续命中
getAge({age: 18})
getAge({age: 20})

```

> **面试一句话总结**：隐藏类优化对象结构，内联缓存优化属性读取，二者配合让弱类型JS拥有接近强类型语言的性能。

# 六、TurboFan JIT 优化编译器

## 1\. V8 编译架构

- **Ignition 解释器**：快速生成字节码，启动快

- **TurboFan 优化编译器**：热点代码编译为**机器码**，极速执行

## 2\. 什么时候触发 TurboFan？

函数、循环被**高频执行（热点代码）**，触发 JIT 编译优化

## 3\. TurboFan 核心优化能力

- 常量折叠、死代码删除

- 函数内联、变量提升优化

- 类型特化：稳定类型直接按机器码运行

- 去冗余计算

## 4\. 反优化（Deoptimization）

如果代码类型突然不稳定、结构突变，TurboFan 会**退回解释执行**，性能暴跌。

对应开发规范：**保证变量类型稳定、对象结构稳定**。
