# 虚拟DOM \+ Diff算法 核心面试知识点（含案例）

# 一、虚拟DOM（Virtual DOM）核心本质

## 1\. 定义本质

**本质**：用纯JS对象，模拟、描述真实DOM节点的层级、属性、内容结构，是真实DOM的内存映射副本。

核心逻辑：操作JS对象成本极低，操作真实DOM重排重绘成本极高，框架通过操作虚拟DOM，批量计算变更后，最小化更新真实DOM。

## 2\. 极简代码案例（手写虚拟DOM）

```javascript
// 真实DOM：<div class="box"><p>文本</p></div>
// 对应虚拟DOM JS对象
const vDom = {
  tag: 'div', // 标签名
  props: {class: 'box'}, // 属性
  children: [
    {tag: 'p', props: {}, children: ['文本']}
  ],
  key: 1 // 节点唯一标识
}
```

## 3\. 虚拟DOM完整工作流程

1. 数据变更 → 生成全新虚拟DOM树

2. 新旧虚拟DOM对比 → **Diff算法**找出差异节点

3. 只将差异部分更新到真实DOM → 减少DOM操作

# 二、Diff算法通用核心规则 \+ Vue3专属Diff策略

## 1\. Diff通用底层规则：同层比较（所有框架通用）

**核心规则**：Diff只会对比同一层级虚拟节点，**绝不跨层级移动、复用节点**。

原因：跨层级DOM移动成本远高于销毁重建，框架直接放弃跨层级复用，优化性价比最高。

### 反例案例（跨层级直接销毁重建）

```html
<!-- 旧结构 -->
<div><p>内容</p></div>
<!-- 新结构：p标签跨层级到外层，直接销毁旧p、新建p，不会移动复用 -->
<p>内容</p><div></div>
```

## 2\. Vue3三大优化Diff策略（面试高频）

### 策略1：PatchFlag 静态标记优化

**定义**：Vue3编译阶段给模板节点打补丁标记，区分**静态节点**和**动态节点**

- 静态节点：文本、固定标签，数据永远不会变更，Diff阶段直接跳过对比

- 动态节点：绑定变量、绑定事件、插值语法节点，仅对比这类节点

#### 案例对比

```html
<template>
  <div>固定静态标题</div> <!-- 静态节点，PatchFlag标记：无更新，Diff跳过 -->
  <div>{{ msg }}</div> <!-- 动态节点，仅Diff此节点 -->
</template>
```

💡 对比Vue2：Vue2需要遍历全部子节点，PatchFlag大幅减少Diff遍历次数，性能大幅提升。

### 策略2：最长递增子序列（核心数组节点优化）

**使用场景**：列表增删、排序、移动场景

**作用**：找出旧列表中**顺序无需变动、可直接复用**的节点，最大化复用旧DOM节点，减少新增/销毁操作。

#### 通俗案例

旧节点key序列：\[2,3,4,5\]，新节点key序列：\[3,4,5,1\]

最长递增子序列：\[3,4,5\] → 这三个节点位置不变，直接复用；仅移动新增1节点，不销毁重建3/4/5节点。

💡 对比Vue2：Vue2仅做首尾优化，复杂排序节点复用率极低，Vue3靠该算法极致优化列表Diff。

## 3\. 补充了解：React Fiber架构

- 出现原因：React传统同步Diff，大列表Diff耗时过长，阻塞主线程，造成页面卡顿

- 核心能力：将虚拟DOM树拆分碎片化任务，利用浏览器空闲时间分片执行Diff

- 核心特性：可中断、可恢复、可优先级调度Diff任务，避免主线程长时间阻塞

- 考点定位：仅了解即可，无需手写，区别于Vue同步执行Diff机制

# 三、Key 的核心作用 \+ 禁止用index做key完整解析

## 1\. Key核心作用

Diff通过**key唯一标识**匹配新旧虚拟节点，判断：节点是否可复用、是否需要更新、是否销毁新建，是节点复用的唯一身份ID。

## 2\. 为什么index下标不能作为列表key（高频面试\+实操坑点）

### 底层问题

数组**头部/中间增删数据**时，数组下标index会重新排序，新旧节点index错乱，Diff错误复用节点，导致：数据绑定错乱、表单值丢失、视图渲染错误、性能损耗。

### 实操踩坑案例（最经典头部新增）

```html
<!-- 初始列表 -->
index:0 文本:A
index:1 文本:B

<!-- 在数组头部新增C，列表变为 -->
index:0 文本:C
index:1 文本:A
index:2 文本:B

```

1. Diff匹配逻辑：新index0匹配旧index0，复用旧A节点，赋值C

2. 新index1匹配旧index1，复用旧B节点，赋值A

3. 额外新建B节点，节点完全没有按需复用，发生**错误复用**

### 3\. key最佳使用规范

- 最优：使用后端返回**唯一稳定id**（id/uuid）

- 特例：列表仅渲染展示、无增删排序、无表单绑定，可临时用index

# 四、虚拟DOM 优势 \+ 局限（面试简答题必背）

## ✅ 核心优势

1. **性能优化**：减少高频真实DOM操作，批量局部更新DOM，避免全局重绘重排

2. **跨平台**：虚拟DOM是JS对象，可适配浏览器DOM、小程序、原生APP（UniApp/ReactNative）

3. **编程体验**：开发者只需修改数据，框架自动更新视图，实现数据视图双向解耦

4. **差异化更新**：Diff算法精准比对，只更新差异节点

## ❌ 原生局限

1. **极小静态页面反而损耗性能**：简单页面，直接操作DOM比虚拟DOM\+Diff更快，多了一层JS比对开销

2. **内存开销**：需要同时存储新旧两棵虚拟DOM树，占用JS内存

3. **无法规避高频DOM操作**：海量高频动画DOM变更，虚拟DOM优化效果有限

# 五、Vue2 VS Vue3 Diff 极简对比（总结背诵）

|版本|Diff优化方案|能力特点|
|---|---|---|
|Vue2|同层比较 \+ 首尾指针优化|全节点遍历对比，复杂列表复用差，无静态节点优化|
|Vue3|同层比较 \+ PatchFlag \+ 最长递增子序列|跳过静态节点，排序列表节点最大化复用，Diff效率极高|
