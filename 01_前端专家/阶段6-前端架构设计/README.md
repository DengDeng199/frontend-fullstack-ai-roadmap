## 学习内容

### 6.1 设计模式在前端中的应用

| 模式 | 应用场景 | 示例 |
|------|---------|------|
| 策略模式 | 表单验证规则、不同场景的算法切换 | 验证规则可配置化 |
| 观察者模式 | 跨组件通信、事件系统 | EventEmitter / mitt |
| 工厂模式 | 组件动态创建、适配器封装 | 动态表单组件 |
| 装饰器模式 | 功能增强（日志/权限/缓存） | 装饰器函数 |
| 代理模式 | 懒加载、缓存代理 | 图片懒加载代理 |
| 单例模式 | 全局状态、WebSocket 连接 | Pinia Store |
| 适配器模式 | 第三方库统一接口 | 多地图 API 适配 |
| 模板方法模式 | 流程标准化 | 表单提交流程 |

### 6.2 组件设计模式

- **Renderless Component（无渲染组件）**：
  - 逻辑与 UI 分离
  - Vue3 的 composable 函数化
  - 场景：复用逻辑但 UI 不同
- **HOC（高阶组件）**：
  - React 模式，Vue 中使用较少
  - 了解即可，知道优劣
- **Composable（组合式函数）**：
  - Vue3 推荐的复用模式
  - useXxx 命名规范
  - 响应式状态管理
- **控制反转（IoC）**：
  - 通过插槽/props 传递渲染逻辑
  - 提高组件灵活性

### 6.3 状态管理架构

- **Pinia 深入**：
  - Store 设计原则
  - 模块化拆分策略
  - 与 TypeScript 深度集成
- **状态分形（Fractal State）**：
  - 组件级状态 vs 全局状态
  - 状态就近原则
  - 何时需要全局状态
- **状态机（XState）**：
  - 有限状态机概念
  - 复杂交互的状态管理（如表单流程、权限流程）
  - Vue3 集成

### 6.4 微前端

- **微前端解决的问题**：巨石前端、多团队协作、技术栈统一
- **qiankun**：
  - 注册/加载/卸载生命周期
  - HTML Entry 接入方式
  - 沙箱隔离（JS 沙箱 / CSS 沙箱）
  - 通信方案（props / CustomEvent / Global State）
- **Module Federation（Webpack 5）**：
  - 远程模块加载
  - 共享依赖
- **微前端选型对比**：
  - qiankun / Module Federation / single-spa / iframe
  - 各方案的优劣和适用场景
- **微前端最佳实践**：
  - 公共依赖抽取
  - 样式隔离
  - 路由管理

### 6.5 设计系统搭建

- **Design Token**：
  - 颜色 / 字体 / 间距 / 圆角 / 阴影
  - CSS 变量管理
  - 多主题支持（亮色/暗色/行业定制）
- **原子组件库**：
  - 按钮 / 输入框 / 选择器 / 表格 / 弹窗等
  - 组件 API 设计原则（一致性/灵活性/可预测性）
- **文档生成**：
  - Storybook 组件文档
  - VitePress 技术文档站
- **版本管理与发布**：
  - Changeset 自动化
  - npm 私有源

### 6.6 技术方案文档撰写

- **ADR（Architecture Decision Records）**：
  - 上下文（Context）
  - 决策（Decision）
  - 状态（Proposed / Accepted / Deprecated）
  - 后果（Consequences）
- **RFC 文档**：
  - 动机 / 详细设计 / 替代方案 / 缺点 / 采纳策略
- **技术选型方法论**：
  - 需求 → 候选方案 → 评估矩阵 → 决策 → POC 验证

### 6.7 前端安全体系

- **CSP 策略**：Content-Security-Policy 配置
- **依赖审计**：npm audit / Snyk / Socket
- **供应链安全**：lock 文件、依赖锁定、签名验证
- **代码安全**：
  - XSS 防御（DOMPurify）
  - CSRF 防御
  - 敏感信息不硬编码

### 6.8 性能预算与长期维护

- **性能预算机制**：
  - JS bundle 大小限制（如 < 300KB gzip）
  - LCP 目标（如 < 2.5s）
  - CI 中集成 bundlesize 检查
- **技术债务管理**：
  - 技术债务记录与评估
  - 定期重构节奏
- **渐进增强**：
  - 核心功能不依赖 JS
  - 浏览器兼容性策略
